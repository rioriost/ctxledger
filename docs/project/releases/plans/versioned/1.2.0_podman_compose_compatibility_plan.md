# `ctxledger` `1.2.0` Implementation Plan — Podman Compose Compatibility

## 1. Purpose

`1.2.0` is a focused infrastructure-quality release.

The goal is to make the `ctxledger` Compose stack run cleanly under
**rootless `podman` / `podman compose`** without breaking the existing
`docker compose` UX, by removing rootless-hostile defaults and quoting
fragility that surfaced when an external project tried to deploy
`ctxledger` with Podman.

This is **not** a runtime, memory, or rules release. No changes to MCP tool
surface, schema, or canonical truth boundary are in scope.

---

## 2. Why `1.2.0` is needed

A separate project attempted to bring up `docker/docker-compose.yml` plus
`docker/docker-compose.small-auth.yml` under `podman compose` and surfaced
ten concrete compatibility / hardening issues, ordered below by severity.

The current Compose stack works under Docker Desktop on `localhost`, but it
implicitly relies on:

- a writable bind mount of the entire repository (`..:/app`)
- runtime `pip install` in the auth-small container
- complex inline shell with double-escaped variables in the private-init
  service
- `python` urllib for healthchecks even when better tools exist in the image
- hardcoded `localhost` for TLS SAN, Grafana root URL, and host port binds
- hardcoded `container_name` values that conflict on multi-instance hosts

Each of these is solvable without breaking the small/local UX. `1.2.0`
performs that cleanup as one bounded, semver-minor release.

---

## 3. Scope

### In scope

1. Compose file hardening for rootless Podman:
   - remove `..:/app` RW bind mount from the production code path
   - keep the bind mount available for live-edit dev workflow via
     `profiles: [dev]` (or an explicit `docker-compose.dev.yml` overlay)
2. Pre-built image for the small auth proxy:
   - new `docker/auth_small/Dockerfile`
   - `auth-small` service consumes the built image with a static `command`
3. Private-init script extraction:
   - move the `sh -lc '...'` payload into
     `scripts/private_init_entrypoint.sh`
   - `command:` becomes a single-element list pointing at the script
4. Healthcheck cleanup:
   - replace `python -c "import urllib.request..."` with `wget`/`curl` where
     the base image already provides one
   - replace `ps aux | grep '[t]raefik'` with `traefik healthcheck --ping`
     and `--ping=true` on the static config (already enabled)
5. Host / bind-host extraction (env-driven, default-preserving):
   - `CTXLEDGER_PUBLIC_HOST` (default `localhost`) used by `GF_SERVER_DOMAIN`,
     `GF_SERVER_ROOT_URL`, README guidance, and TLS SAN documentation
   - `CTXLEDGER_BIND_HOST` (default empty → all interfaces, matching today)
     applied to host-port mappings (`${CTXLEDGER_BIND_HOST:-}:55432:5432`,
     `:3000:3000`, `:8443:8443`)
6. Remove hardcoded `container_name:` from compose services so the project
   prefix (`name: ctxledger`) provides naming and multiple instances do not
   collide. This is harmless under Docker too.
7. Documentation updates:
   - README (en + ja): add a "Run under Podman" subsection in the small
     Quick Start, document `CTXLEDGER_PUBLIC_HOST` / `CTXLEDGER_BIND_HOST`,
     and note Quadlet as the recommended path for systemd-managed Podman
     deployments
   - CHANGELOG entry under `[1.2.0]`
   - brief operator note on `podman compose` (Compose v2 plugin against the
     Podman socket) being preferred over `podman-compose` (Python) for full
     `condition: service_completed_successfully` support
8. Version bump:
   - `pyproject.toml` `version = "1.2.0"`
   - `CTXLEDGER_APP_VERSION` defaults in `docker-compose.yml` and
     `docker-compose.small-auth.yml` updated to `1.2.0`

### Non-goals

- No changes to MCP tools, schemas, memory model, or rules.
- No new Quadlet unit files shipped in this release. Quadlet is mentioned
  as the recommended systemd-managed path; shipping units is a future
  release.
- No SELinux `:Z` / `:z` labeling for the `..:/app` mount — instead, the
  mount itself is removed from the production path. `:Z` is mentioned only
  in the dev profile documentation as a known requirement on SELinux
  enforcing hosts when the dev overlay is used.
- No change to the auth-small wire protocol or token format.

## 5. Slices (execution order)

### S0 — Dockerfile confirmation (no behavior change)

The current `Dockerfile`:

- `COPY pyproject.toml uv.lock README.md LICENSE ./`
- `COPY src ./src`, `COPY schemas ./schemas`, `COPY scripts ./scripts`
- builds a venv at `/opt/ctxledger-venv` with `uv sync --frozen --active`

This already bakes the application into the image. The `..:/app` bind
mount in compose was therefore masking the baked image at runtime. S0 is
purely to confirm and document that the baked image is the source of truth
and the bind mount in `docker-compose.yml` is unnecessary for production
use — it exists only as an implicit dev convenience.

No code change in S0; this is the justification record for S1.

### S1 — Compose hardening (`docker-compose.yml`, `docker-compose.small-auth.yml`)

- Remove `volumes: - ..:/app` from `ctxledger`, `ctxledger-private`, and
  `ctxledger-private-init`.
- Remove `container_name:` from `postgres`, `ctxledger`,
  `ctxledger-private`, `ctxledger-private-init`, `ctxledger-auth-small`,
  `ctxledger-traefik`, `ctxledger-grafana`. Compose project name
  (`name: ctxledger`) provides the prefix.
- `working_dir: /app` is preserved (matches the image layout).

### S2 — `auth-small` pre-built image

- New `docker/auth_small/Dockerfile`:
  - `FROM python:3.14-slim`
  - install `fastapi` and `uvicorn[standard]` at build time using `uv` or
    plain `pip` (single layer, no network at runtime)
  - `COPY docker/auth_small/src /app/src`
  - `WORKDIR /app`
  - `EXPOSE 8090`
  - `CMD ["python", "-m", "uvicorn", "auth_small_app:app", "--app-dir", "/app/src", "--host", "0.0.0.0", "--port", "8090"]`
- `auth-small` service in `docker-compose.small-auth.yml`:
  - replace `image: python:3.14-slim` with `build:` block + `image:
    ctxledger-auth-small:local`
  - drop the `sh -lc` runtime install command; rely on Dockerfile `CMD`
  - drop the read-only `./auth_small/src:/app/src:ro` bind mount (the
    sources are baked into the image; for dev overlay this returns)

### S3 — Private-init script extraction

- New `scripts/private_init_entrypoint.sh` (POSIX `sh`, `set -eu`):
  - reads `CTXLEDGER_PRIVATE_INIT_REBUILD_AGE_GRAPH` and
    `CTXLEDGER_PRIVATE_INIT_BOOTSTRAP_AGE_GRAPH` from the environment
    directly (no `$${VAR}` double-escaping)
  - resolves the AGE bootstrap command into a shell function rather than a
    string, so the third `sh -lc` is removed
  - runs `apply-schema`, `ensure_age_extension.py`, the AGE bootstrap
    branch, and the optional `setup_grafana_observability.py` step in
    sequence with explicit exit semantics
- `ctxledger-private-init` service `command:` becomes
  `["scripts/private_init_entrypoint.sh"]`
- mark the script executable in the image (`RUN chmod +x` in `Dockerfile`
  already covers `scripts/*.sh` for the two existing scripts; extend the
  list)

### S4 — Healthcheck cleanup

- `postgres`: keep `pg_isready` (already correct)
- `ctxledger`, `ctxledger-private`: replace urllib healthcheck with
  `wget -q -O /dev/null http://127.0.0.1:8080/debug/runtime || exit 1`.
  The base image `python:3.14-slim` does not ship `wget` by default; the
  `Dockerfile` already installs `curl`, so use `curl -fsS
  http://127.0.0.1:8080/debug/runtime > /dev/null` instead.
- `auth-small`: same pattern with `curl` (will be available in the new
  `docker/auth_small/Dockerfile` — add `curl` to the apt install list).
- `traefik`: replace `ps aux | grep` with
  `["CMD", "traefik", "healthcheck", "--ping"]` (image already configured
  with `--ping=true` in static args).
- `grafana`: `wget` is present in the official image; keep current.

### S5 — Host / bind-host variables

- Introduce two new env knobs, both with backward-compatible defaults:
  - `CTXLEDGER_PUBLIC_HOST` (default `localhost`)
  - `CTXLEDGER_BIND_HOST` (default empty)
- Apply to:
  - `GF_SERVER_DOMAIN: ${CTXLEDGER_GRAFANA_DOMAIN:-${CTXLEDGER_PUBLIC_HOST:-localhost}}`
  - `GF_SERVER_ROOT_URL: ${CTXLEDGER_GRAFANA_ROOT_URL:-http://${CTXLEDGER_PUBLIC_HOST:-localhost}:3000}`
  - host port mappings:
    - postgres: `"${CTXLEDGER_BIND_HOST:-}:55432:5432"` (or split
      `${CTXLEDGER_BIND_HOST:+${CTXLEDGER_BIND_HOST}:}55432:5432` if the
      empty-prefix form needs a guard under podman-compose)
    - grafana: `"${CTXLEDGER_BIND_HOST:-}:3000:3000"`
    - traefik: `"${CTXLEDGER_BIND_HOST:-}:8443:8443"`
- README adds a short "deploy on a non-localhost host" note pointing at
  these variables and the `mkcert` SAN list.

### S6 — Dev overlay for live-edit workflow

- New `docker/docker-compose.dev.yml` overlay re-introduces the
  `..:/app:Z` bind mount for `ctxledger` and `ctxledger-private`, plus the
  `./auth_small/src:/app/src:ro,Z` bind mount for `auth-small`.
- README documents the explicit overlay invocation:
  ```
  docker compose -f docker/docker-compose.yml \
                 -f docker/docker-compose.small-auth.yml \
                 -f docker/docker-compose.dev.yml up --build
  ```
- The `:Z` label is documented as required only on SELinux enforcing
  hosts. Docker Desktop ignores it.

### S7 — Documentation and version bump

- `README.md` and `README.ja.md`:
  - add "Run under Podman" subsection inside the small Quick Start, with
    the recommendation to use `podman compose` (Compose v2 + Podman
    socket) for full `condition: service_completed_successfully` support,
    and `podman-compose` (Python) listed as a fallback with caveats
  - document `CTXLEDGER_PUBLIC_HOST` and `CTXLEDGER_BIND_HOST`
  - document the new dev overlay
  - mention Quadlet (`.container` / `.kube` / `.pod`) as the recommended
    path for systemd-managed Podman deployments, with the note that
    `restart:` policies in compose can be replaced by `Restart=always` in
    a Quadlet unit
- `docs/project/releases/CHANGELOG.md`:
  - add `## [1.2.0]` section describing the rootless-podman compatibility
    work, the `auth-small` image, the script extraction, the new env
    knobs, and the removal of `container_name:` and the production
    `..:/app` bind mount
- `pyproject.toml`: bump `version` to `1.2.0`
- `docker-compose.yml` and `docker-compose.small-auth.yml`:
  `CTXLEDGER_APP_VERSION: 1.2.0`

---

## 6. Backward compatibility

- Default behavior on `localhost` with Docker Desktop is preserved:
  - port mappings remain `55432`, `3000`, `8443`
  - Grafana root URL still defaults to `http://localhost:3000`
  - small auth token env var name unchanged
  - TLS cert paths unchanged
- Live-edit workflow is preserved through the explicit dev overlay
  instead of the implicit RW bind mount.
- `condition: service_completed_successfully` is retained; operators on
  older `podman-compose` are pointed at `podman compose` (Compose v2).

---

## 7. Verification

Acceptance for `1.2.0` requires the following to all hold on a clean host:

1. `docker compose -f docker/docker-compose.yml -f docker/docker-compose.small-auth.yml config` succeeds with no warnings.
2. `podman compose -f docker/docker-compose.yml -f docker/docker-compose.small-auth.yml config` succeeds with no warnings on a Podman 5.x host with the Compose v2 plugin.
3. The small stack starts end-to-end under both `docker compose` and
   `podman compose` without:
   - a writable bind mount of the repository
   - any container performing `pip install` at runtime
4. `scripts/mcp_http_smoke.py --base-url https://localhost:8443
   --bearer-token <token> --scenario workflow --workflow-resource-read
   --insecure` passes against both runtimes.
5. Grafana `http://localhost:3000` returns 200 from `/api/health`.
6. Setting `CTXLEDGER_BIND_HOST=127.0.0.1` produces port mappings that
   bind only on loopback (verified via `ss -lntp` or `lsof -iTCP`).
7. Setting `CTXLEDGER_PUBLIC_HOST=ctxledger.lan` reflects in
   `GF_SERVER_ROOT_URL` inside the running grafana container.
8. The dev overlay still mounts the repository RW for live editing and
   still works under Docker Desktop.

---

## 8. Risks

- `podman-compose` (Python) variable interpolation differences may
  require a `${CTXLEDGER_BIND_HOST:+${CTXLEDGER_BIND_HOST}:}PORT:PORT`
  form instead of the cleaner `${CTXLEDGER_BIND_HOST:-}:PORT:PORT`. The
  cleaner form is preferred and validated under `docker compose` and
  `podman compose`; the guarded form is the documented fallback.
- Removing `container_name:` changes container names from
  `ctxledger-postgres` to `ctxledger-postgres-1`. Operator scripts that
  hardcoded the old name must use `docker compose exec postgres ...`
  instead. README documents this.
- The `auth-small` image must be rebuilt when `auth_small_app.py`
  changes. The dev overlay restores the source bind mount to make this
  cheap during development.

---

## 9. Out of scope, deferred

- Shipping Quadlet unit files (`.container`, `.kube`, `.pod`).
- Replacing Traefik with a different proxy.
- Changing the auth-small protocol or adding a non-static-token mode.
- Changing how `mkcert` is invoked; only the SAN guidance is updated.
