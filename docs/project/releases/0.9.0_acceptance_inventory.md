## Status value meanings

This inventory uses:

- `pass`
- `partial`
- `deferred`

Read them as:

- `pass`
  - the bounded slice exists strongly enough that the repository can already
    claim it for the current milestone reading
- `partial`
  - meaningful implementation exists, but the acceptance item is not strong
    enough to claim complete closeout yet
- `deferred`
  - intentionally outside the current completed slice
